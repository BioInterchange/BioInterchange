Genomic Feature and Variation Ontology — GFVO

Detailed description to follow soon... GFVO makes use of FALDO and VariO.

Examples

Example 1: Loci and Basic Genomic Feature Annotations

Encoding loci and basic genomic feature annotations. The example covers lines 0-3 of the example gene description given in the GFF3 specification.

@prefix : <http://www.biointerchange.org/gfvo#> .
@prefix dc: <http://purl.org/dc/terms/> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix xml: <http://www.w3.org/XML/1998/namespace> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix faldo: <http://biohackathon.org/resource/faldo#> .
@prefix vario: <http://purl.obolibrary.org/obo/> .
@base <http://www.biointerchange.org/gfvo#> .

#################################################################
#
#    Example 1
#
#    Encoding loci and basic genomic feature annotations. The
#    example covers lines 0-3 of the example gene description
#    given in the GFF3 specification.
#           (http://www.sequenceontology.org/resources/gff3.html)
#
#    Please note that this seemingly verbose data representation
#    is slighly misleading. The reuse of non-literals as well as
#    binary RDF encodings such as RDF/HDT permits a more concise
#    representation of the same data.
#
#################################################################

:ExampleSet1 rdf:type :File ,
                      owl:NamedIndividual ;
             :gffVersion "3" ;
             rdfs:comment "A simple example of a hierarchical genomic feature dependency. This example is an excerpt of another example given in the GFF3 specification (http://sequenceontology.org/resources/gff3.html)."@en .

:ExampleSet1ChromEnd rdf:type faldo:ExactPosition ,
                              owl:NamedIndividual ;
                     faldo:position "1497228" .

:ExampleSet1ChromStart rdf:type faldo:ExactPosition ,
                                owl:NamedIndividual ;
                       faldo:position "1" .

:ExampleSet1Feature1 rdf:type :Feature ,
                              owl:NamedIndividual ;
                     :id "gene00001" ;
                     :name "EDEN" ;
                     :type "http://www.sequenceontology.org/browser/current_release/term/SO:0000704" ;
                     :seqid :ExampleSet1Landmark1 .

:ExampleSet1Feature1_2Start rdf:type faldo:ExactPosition ,
                                     owl:NamedIndividual ;
                            faldo:position "1000" .

:ExampleSet1Feature2 rdf:type :Feature ,
                              owl:NamedIndividual ;
                     :id "tfbs00001" ;
                     :type "http://www.sequenceontology.org/browser/current_release/term/SO:0000235" ;
                     :parent :ExampleSet1Feature1 ;
                     :locus :ExampleSet1Region3 .

:ExampleSet1Feature2End rdf:type faldo:ExactPosition ,
                                 owl:NamedIndividual ;
                        faldo:position "1012" .

:ExampleSet1Landmark1 rdf:type :Landmark ,
                               owl:NamedIndividual ;
                      :id "ctg123" ;
                      :locus :ExampleSet1Region1 .

:ExampleSet1Region1 rdf:type faldo:Region ,
                             owl:NamedIndividual ;
                    faldo:begin :ExampleSet1ChromEnd ,
                                :ExampleSet1ChromStart .

:ExampleSet1Region2 rdf:type faldo:ForwardStrandPosition ,
                             owl:NamedIndividual ;
                    faldo:begin :ExampleSet1Feature1_2Start ;
                    faldo:end :ExampleSet1Feature1End .

:ExampleSet1Region3 rdf:type faldo:Region ,
                             owl:NamedIndividual ;
                    faldo:begin :ExampleSet1Feature1_2Start ;
                    faldo:end :ExampleSet1Feature2End .

:ExampleSet1Feature1End rdf:type faldo:ExactPosition ,
                                owl:NamedIndividual ;
                       faldo:position "9000" .

Example 2: Encoding of a Sequence Alignment

Encoding of a sequence alignment. This is part of the GFF3 specification, denoting the alignment between the reference sequence "chr3" and the target sequence "EST23".

@prefix : <http://www.biointerchange.org/gfvo#> .
@prefix dc: <http://purl.org/dc/terms/> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix xml: <http://www.w3.org/XML/1998/namespace> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix faldo: <http://biohackathon.org/resource/faldo#> .
@prefix vario: <http://purl.obolibrary.org/obo/> .
@base <http://www.biointerchange.org/gfvo#> .

#################################################################
#
#    Example 2
#
#    Encoding of a sequence alignment. This is part of the GFF3
#    specification, denoting the alignment between the reference
#    sequence "chr3" and the target sequence "EST23".
#           (http://www.sequenceontology.org/resources/gff3.html)
#
#################################################################

:ExampleSet2 rdf:type :Set ,
                      owl:NamedIndividual ;
             rdfs:comment "An example of a sequence alignment. This example is an excerpt of another example given in the GFF3 specification (http://sequenceontology.org/resources/gff3.html)."@en .

:ExampleSet2Feature1 rdf:type :Feature ,
                              owl:NamedIndividual ;
                     :id "Match1" ;
                     :type "http://www.sequenceontology.org/browser/current_release/term/SO:0000343" ;
                     :seqid :ExampleSet2Landmark1 .

:ExampleSet2Feature2 rdf:type :Feature ,
                              owl:NamedIndividual ;
                     :id "EST23" ;
                     :type "http://www.sequenceontology.org/browser/current_release/term/SO:0000345" ;
                     :target :ExampleSet2Target1 .

:ExampleSet2Landmark1 rdf:type :Landmark ,
                               owl:NamedIndividual ;
                      :id "chr3" .

:ExampleSet2Target1 rdf:type :Target ,
                             owl:NamedIndividual ;
                    :alignment :ExampleSet2AlignmentOperation1 .

:ExampleSet2AlignmentOperation1 rdf:type :AlignmentOperation ,
                                owl:NamedIndividual ;
                       :operation :Match ;
                       :span "8" ;
                       :first :ExampleSet2AlignmentOperation1 ;
                       :rest :ExampleSet2AlignmentOperation2 .

:ExampleSet2AlignmentOperation2 rdf:type :AlignmentOperation ,
                                owl:NamedIndividual ;
                       :operation :TargetSequenceGap ;
                       :span "3" ;
                       :first :ExampleSet2AlignmentOperation2 ;
                       :rest :ExampleSet2AlignmentOperation3 .

:ExampleSet2AlignmentOperation3 rdf:type :AlignmentOperation ,
                                owl:NamedIndividual ;
                       :operation :Match ;
                       :span "6" ;
                       :first :ExampleSet2AlignmentOperation3 ;
                       :rest :ExampleSet2AlignmentOperation4 .

:ExampleSet2AlignmentOperation4 rdf:type :AlignmentOperation ,
                                owl:NamedIndividual ;
                       :operation :ReferenceSequenceGap ;
                       :span "1" ;
                       :first :ExampleSet2AlignmentOperation4 ;
                       :rest :ExampleSet2AlignmentOperation5 .

:ExampleSet2AlignmentOperation5 rdf:type :AlignmentOperation ,
                                owl:NamedIndividual ;
                       :operation :Match ;
                       :span "6" ;
                       :first :ExampleSet2AlignmentOperation5 .

Ontology Reference

Ontology namespace "gfvo": http://www.biointerchange.org/gfvo#

Classes

Class gfvo:Alignment
Subclass of: GenomicEnumeration
Individuals: ForwardReferenceSequenceFrameshift, Match, ReferenceSequenceGap, ReverseReferenceSequenceFrameshift, TargetSequenceGap
Label: Alignment
Comment: An alignment captures the matches, gaps or frameshifts between a reference sequence and a target sequence (see "The Gap Attribute", http://sequenceontology.org/resources/gff3.html). The reference is typically a Feature class instance that is linked to a target sequence via a Target class instance.
GFF3GVF

Class gfvo:AlignmentOperation
Subclass of: LocusAnnotation
Label: Alignment Operation
Comment: An alignment operation captures the type of alignment (see Alignment enumeration class) and alignment length between a reference sequence and target sequence. This relationship is typically expressed between a Feature class instance and a Target class instance. Note that an Alignment Operation is a list, where the order of the alignment operations is of significance.
GFF3GVF

Class gfvo:Attribute
Subclass of: CustomAnnotation
Label: Attribute
Comment: An attribute represents a tag/value pair that is not covered by either GFF3, GTF, GVF or VCF specification. For example, lowercase tags in GFF3 are permitted to allow data providers to provide additional information about a genomic feature, variant, or its meta-data. Tag/value pair attributes that are within the aforementioned specifications are expressed using classes such as Variant, Breakpoint or Technology Platform.
GFF3GTFGVFVCF

Class gfvo:Breakpoint
Subclass of: LocusAnnotation
Label: Breakpoint
Comment: A breakpoint describes the source or destination of a zero-length sequence alteration. These alterations are typically insertions, deletions or translocations according to the GVF specification (see "Breakpoint_detail" in http://sequenceontology.org/resources/gvf.html).
GVF

Class gfvo:Chromosome
Subclass of: LocusAnnotation
Label: Chromosome
Comment: A chromosome is an abstract representation of an unnamed chromosome to represent ploidy within a data set. A Chromosome is typically used in conjunction with Sequenced Individual instances. For placing genomic features (Feature class instances) on a chromosome, contig, scaffold, etc., please see the Landmark class.
GFF3GVF

Class gfvo:CustomAnnotation
Subclass of: FeatureAnnotation
Label: Custom Annotation
Comment: A custom annotation is any additionally provided genomic annotation that is not covered by the GFF3, GTF, GVF or VCF specifications, and hence, not represented by any other class in the ontology. For example, in GFF3, this would typically cover the attributes with lowercase tags of column 9, which are permitted by the standard as user defined annotations.

Class gfvo:DataSource
Subclass of: TechnologicalAnnotation
Label: Data Source
Comment: Provides information about the source of the data. For example, it can link out to actual sequences associated with Feature individuals in a Set.
GVF

Class gfvo:DataType
Subclass of: TechnologicalEnumeration
Individuals: ArrayComparativeGenomicHybridization, DNAMicroarray, DNASequence, RNASequence
Label: Data Type
Comment: Determines the physical type of data behind a variant sequence. For example, RNA Sequence. The data type enumeration is used by the Data Source class.
GVF

Class gfvo:Effect
Subclass of: VariationAnnotation
Label: Effect
Comment: Describing the effect of a feature variant.
GVF

Class gfvo:Enumeration
Label: Enumeration
Comment: The enumeration class is providing a finite and complete set of values (OWL individuals). This class has no immediate individuals associated with it, but instead indicates that its children (Genomic Enumeration, Sequenced Individual Enumeration and Technological Enumeration) are enumerations as described in more detail in the Protege OWL Tutorial (http://owl.cs.manchester.ac.uk/tutorials/protegeowltutorial/resources/ProtegeOWLTutorialP4_v1_1.pdf).

Class gfvo:Feature
Subclass of: Record
Label: Feature
Comment: The feature class captures information about genomic sequence features. A genomic feature can be a large object, such as a chromosome or contig, down to single base-pair reference or variant alleles. A feature class instance is a composite of further information that is encoded using Attribute, Reference, "Variation" and other sub-classes of Feature Annotation.
GFF3GTFGVFVCF

Class gfvo:FeatureAnnotation
Subclass of: Record
Label: Feature Annotation
Comment: A feature annotation is a class whose instances are used as composites in conjunction with Feature class instances. The Feature Annotation class is typically not used as type annotation. It rather denotes that its subclasses are to be used for extending genomic features with additional information.

Class gfvo:File
Subclass of: Set
Label: File
Comment: A file represents the contents of a GFF3, GTF, GVF or VCF file. It can capture genomic meta-data that is specific to any of these file formats. The result of unions, intersections or other operations between File class instances should be capture with the generic Set class, which is format independent.
GFF3GTFGVFVCF

Class gfvo:GenomicEnumeration
Subclass of: Enumeration
Label: Genomic Enumeration
Comment: A genomic enumeration provides a finite choice between OWL individuals that are denoting instances of genomic concepts. For example, Zygosity.

Class gfvo:GenomicSource
Subclass of: GenomicEnumeration
Individuals: Germline, Prenatal, Somatic
Label: Genomic Source
Comment: An enumerated class for determining the genomic source (cell type) of sequenced data.
GVF

Class gfvo:Landmark
Subclass of: LocusAnnotation
Label: Landmark
Comment: A landmark establishes a coordinate system for features. Landmarks can be chromosomes, contigs, scaffolds or other constructs that can harbor Feature class instances. For expressing ploidy within a data set, please refer to the Chromosome class.
GFF3GTFGVFVCF

Class gfvo:LocusAnnotation
Subclass of: FeatureAnnotation
Label: Locus Annotation
Comment: A locus annotation puts a genomic feature or variation in context or provides additional information about particular location related aspects that fall within a feature's range (expressed by the locus object property of the Feature class). The Locus Annotation class is not instantiated itself, but rather one of its subclasses. For example, Landmark is used to place a Feature on a chromosome, scaffold, contig, or other genomic entity.

Class gfvo:Method
Subclass of: TechnologicalAnnotation
Label: Method
Comment: Information about the used scoring algorithm or method.
GVF

Class gfvo:PhenotypeDescription
Subclass of: SetAnnotation
Label: Phenotype Description
Comment: A phenotype description represents additional information about a sequenced individual's phenotype.
GVF

Class gfvo:ReadType
Subclass of: TechnologicalEnumeration
Individuals: Fragment, Pair
Label: Read Type
Comment: Type of reads obtained for a given technology platform. Reads can be either Fragment or Pair by this enumeration.
GVF

Class gfvo:Record
Label: Record
Comment: A record is an abstract representation of a genomic feature, genomic variation, or its composite elements. A record is typically not instantiated in itself, but rather its child classes (s.a. Feature, "Variation", Reference, etc.) are types for OWL individuals.

Class gfvo:Reference
Subclass of: FeatureAnnotation
Label: Reference
Comment: Denotes the reference sequence of a feature. The reference sequence is of importance when dealing with genomic variation data, which is expressed by the Variant class.
GVFVCF

Class gfvo:SequencedIndividual
Subclass of: FeatureAnnotation
Label: Sequenced Individual
Comment: Aggregated sequencing information for a particular individual.
GVF

Class gfvo:SequencedIndividualEnumeration
Subclass of: Enumeration
Label: Sequenced Individual Enumeration
Comment: A sequenced individual enumeration provides a finite choice between OWL individuals that are denoting instances that are related to human individuals, patients, donors, etc. For example, Sex is an enumeration related to the sex of sequenced individuals.

Class gfvo:Set
Label: Set
Comment: A set is a container for genomic sequence features and related information that is independent of data provenance. A set may contain information about any genomic features including -- but not limited to -- contents of GFF3, GTF, GVF and VCF files. The latter are better represented by File class instances, whereas the result of unions or intersections between different File class instances should be captured within this format-independent Set class.
GFF3GTFGVFVCF

Class gfvo:SetAnnotation
Subclass of: Record
Label: Set Annotation
Comment: A set annotation denotes that its subclasses can be used to extend a Set with additional information, s.a. a Phenotype Description. Set annotations are file format independent and can be preserved when carrying out operations between File/Set instances. Depending on the annotation, it may or may not be able to carry it over in operations such as taking the union of two or more sets, since the scope of a Set Annotation applies to all Feature instances within the resulting Set.

Class gfvo:Sex
Subclass of: SequencedIndividualEnumeration
Individuals: Female, Hermaphrodite, Male
Label: Sex
Comment: For single individual sets, the Sex class enumeration instances can be used to specify the sex of a sequenced (real-life) individual.
GVF

Class gfvo:StructuredAttribute
Subclass of: CustomAnnotation
Label: Structured Attribute
Comment: A structured attribute denotes a tag/value pair where the value is a composite, but which is not defined in the GVF specification. The GVF specification does not explicitly permit user-defined structured attributes (see "Structured Pragmas" in http://sequenceontology.org/resources/gvf.html), but it is conceivable that an RDFization tool might support this use case. For some loosly defined structured data in GVF, the Structured Attribute class is used as well to capture the non-exhaustive list of possible data assignments of the GVF specification.
GFF3GTFGVFVCF

Class gfvo:Target
Subclass of: LocusAnnotation
Label: Target
Comment: A target expresses the relationship between a Feature instance and an alignment. In GFF3, the alignment can be a nucleotide-to-nucleotide or protein-to-nucleotide (see "The Gap Attribute", http://sequenceontology.org/resources/gff3.html), but this restriction is not enforced here. Note that the object property alignment links out to a list of Alignment Operation class instances, where only the first operation in that list is referenced via the alignment property. The remainder of operations can be accessed by stepping through the list.
GVF

Class gfvo:TechnologicalAnnotation
Subclass of: FeatureAnnotation
Label: Technological Annotation

Class gfvo:TechnologicalEnumeration
Subclass of: Enumeration
Label: Technological Enumeration
Comment: A technical enumeration provides a finite choice between OWL individuals that are denoting instances related to technical aspects of genome sequencing.

Class gfvo:TechnologyPlatform
Subclass of: TechnologicalAnnotation
Label: Technology Platform
Comment: Details about the sequencing/microarray technology used to gather the data in a set.
GVF

Class gfvo:Variant
Subclass of: VariationAnnotation
Label: Variant
Comment: Describing specific sequence alterations of a genomic feature. A variant is related to Reference class instances, which denote the sequence that serves as a basis for sequence alteration comparisons.
GVFVCF

Class gfvo:VariationAnnotation

Datatype Property gfvo:fileDatatypeProperty
Label: file datatype property
Comment: Datatype properties related to File class instances. Subclasses of this property are used for denoting data that is specific to files only and which is only preserved as a means to encode for all data items defined by the GFF3, GTF, GVF and VCF specifications. These properties cannot be carried over into the result of set operations (s.a. union) between File/Set class instances. Keeping these properties in the result set between set operations may result in wrong annotations or contradictions (e.g., mutliple assignment of different versions or dates).
